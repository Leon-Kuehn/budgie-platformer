# Contributing to budgie-platformer

Thanks for your interest in contributing!

## How to contribute

1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit with a clear message: `git commit -m "feat: add your feature"`
5. Push your branch: `git push origin feature/your-feature-name`
6. Open a Pull Request against `main`

## Branch naming

- `feature/` – new features
- `fix/` – bug fixes
- `docs/` – documentation changes
- `chore/` – maintenance tasks

## Commit style

Please use [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `style:` formatting
- `refactor:` code restructuring
- `chore:` maintenance

## Pull Requests

- Keep PRs small and focused on one thing
- Describe what you changed and why
- Make sure all conversations are resolved before merging

## Reporting bugs

Use the [Bug Report issue template](.github/ISSUE_TEMPLATE/bug_report.md).

## Code style

- Use clear, readable variable names
- Comment complex logic
- Keep functions small and focused
